# The Unicode Standard
Unicode is a text encoding standard that maps characters to integer code points. As of Unicode 16.0 (released
in September 2024), the standard defines 154,998 characters across 168 scripts. For example, the character
‚Äús‚Äù has the code point 115 (typically notated as U+0073, where U+ is a conventional prefix and 0073 is 115 in
hexadecimal), and the character ‚ÄúÁâõ‚Äù has the code point 29275. In Python, you can use the ord() function
to convert a single Unicode character into its integer representation. The chr() function converts an integer
Unicode code point into a string with the corresponding character.

```python
>>> ord('Áâõ')
29275
>>> chr(29275)
'Áâõ'
```

## 2.2 Unicode Encodings

**UTF-8** –∏ **UTF-16** ‚Äî —ç—Ç–æ —Å–ø–æ—Å–æ–±—ã –∫–æ–¥–∏—Ä–æ–≤–∞–Ω–∏—è —Å–∏–º–≤–æ–ª–æ–≤ —Å—Ç–∞–Ω–¥–∞—Ä—Ç–∞ **Unicode** –≤ –±–∞–π—Ç—ã. Unicode –Ω—É–∂–µ–Ω, —á—Ç–æ–±—ã –∫–∞–∂–¥–æ–º—É —Å–∏–º–≤–æ–ª—É –∏–∑ –ª—é–±–æ–≥–æ —è–∑—ã–∫–∞ (–¥–∞–∂–µ —ç–º–æ–¥–∑–∏ üòä) –ø—Ä–∏—Å–≤–æ–∏—Ç—å —É–Ω–∏–∫–∞–ª—å–Ω—ã–π –Ω–æ–º–µ—Ä (–Ω–∞–ø—Ä–∏–º–µ—Ä, `A` ‚Üí `U+0041`, `–Ø` ‚Üí `U+042F`). –ù–æ —á—Ç–æ–±—ã —Å–æ—Ö—Ä–∞–Ω–∏—Ç—å –∏–ª–∏ –ø–µ—Ä–µ–¥–∞—Ç—å —Ç–µ–∫—Å—Ç, —ç—Ç–∏ –Ω–æ–º–µ—Ä–∞ –Ω—É–∂–Ω–æ –ø—Ä–µ–≤—Ä–∞—Ç–∏—Ç—å –≤ –±–∞–π—Ç—ã ‚Äî –¥–ª—è —ç—Ç–æ–≥–æ –∏ —Å—É—â–µ—Å—Ç–≤—É—é—Ç UTF-8 –∏ UTF-16.

```python
>>> test_string = "hello! „Åì„Çì„Å´„Å°„ÅØ!"
>>> utf8_encoded = test_string.encode("utf-8")
>>> print(utf8_encoded)
b'hello! \xe3\x81\x93\xe3\x82\x93\xe3\x81\xab\xe3\x81\xa1\xe3\x81\xaf!'
>>> print(type(utf8_encoded))
<class 'bytes'>
>>> # Get the byte values for the encoded string (integers from 0 to 255).
>>> list(utf8_encoded)
[104, 101, 108, 108, 111, 33, 32, 227, 129, 147, 227, 130, 147, 227, 129, 171, 227, 129,
161, 227, 129, 175, 33]
>>> # One byte does not necessarily correspond to one Unicode character!
>>> print(len(test_string))
13
>>> print(len(utf8_encoded))
23
>>> print(utf8_encoded.decode("utf-8"))
hello! „Åì„Çì„Å´„Å°„ÅØ!
```

### **–ß—Ç–æ —Ç–∞–∫–æ–µ UTF-8?**
- **–ü–µ—Ä–µ–º–µ–Ω–Ω–∞—è –¥–ª–∏–Ω–∞**: –°–∏–º–≤–æ–ª—ã –∫–æ–¥–∏—Ä—É—é—Ç—Å—è **–æ—Ç 1 –¥–æ 4 –±–∞–π—Ç–æ–≤**.
  - –õ–∞—Ç–∏–Ω—Å–∫–∏–µ –±—É–∫–≤—ã (A-Z, a-z) ‚Üí **1 –±–∞–π—Ç** (–∫–∞–∫ –≤ —Å—Ç–∞—Ä–æ–º ASCII).
  - –ö–∏—Ä–∏–ª–ª–∏—Ü–∞, –≥—Ä–µ—á–µ—Å–∫–∏–µ —Å–∏–º–≤–æ–ª—ã ‚Üí **2 –±–∞–π—Ç–∞**.
  - –ò–µ—Ä–æ–≥–ª–∏—Ñ—ã, —ç–º–æ–¥–∑–∏ ‚Üí **3-4 –±–∞–π—Ç–∞**.
- **–°–æ–≤–º–µ—Å—Ç–∏–º–æ—Å—Ç—å —Å ASCII**: –§–∞–π–ª—ã, –Ω–∞–ø–∏—Å–∞–Ω–Ω—ã–µ –≤ ASCII, –∞–≤—Ç–æ–º–∞—Ç–∏—á–µ—Å–∫–∏ –≤–∞–ª–∏–¥–Ω—ã –≤ UTF-8.
- **–ü—Ä–∏–º–µ—Ä**: –°–ª–æ–≤–æ `Hello` ‚Üí `48 65 6C 6C 6F` (5 –±–∞–π—Ç–æ–≤), `–ü—Ä–∏–≤–µ—Ç` ‚Üí `D0 9F D1 80 D0 B8 D0 B2 D0 B5 D1 82` (12 –±–∞–π—Ç–æ–≤).

### **–ß—Ç–æ —Ç–∞–∫–æ–µ UTF-16?**
- **–§–∏–∫—Å–∏—Ä–æ–≤–∞–Ω–Ω–∞—è –∏–ª–∏ –ø–µ—Ä–µ–º–µ–Ω–Ω–∞—è –¥–ª–∏–Ω–∞**: 
  - –ë–æ–ª—å—à–∏–Ω—Å—Ç–≤–æ —Å–∏–º–≤–æ–ª–æ–≤ ‚Üí **2 –±–∞–π—Ç–∞** (–Ω–∞–ø—Ä–∏–º–µ—Ä, `A` ‚Üí `0041`, `–Ø` ‚Üí `042F`).
  - –†–µ–¥–∫–∏–µ —Å–∏–º–≤–æ–ª—ã (—ç–º–æ–¥–∑–∏, –¥—Ä–µ–≤–Ω–∏–µ —è–∑—ã–∫–∏) ‚Üí **4 –±–∞–π—Ç–∞** (–∏—Å–ø–æ–ª—å–∑—É—é—Ç—Å—è **—Å—É—Ä—Ä–æ–≥–∞—Ç–Ω—ã–µ –ø–∞—Ä—ã**).
- **–ü–æ—Ä—è–¥–æ–∫ –±–∞–π—Ç–æ–≤**: –í–∞–∂–µ–Ω –ø–æ—Ä—è–¥–æ–∫ –±–∞–π—Ç–æ–≤ (**BOM**, Byte Order Mark: `FEFF` –∏–ª–∏ `FFFE`), —á—Ç–æ –º–æ–∂–µ—Ç —É—Å–ª–æ–∂–Ω—è—Ç—å –æ–±—Ä–∞–±–æ—Ç–∫—É.
- **–ü—Ä–∏–º–µ—Ä**: `Hello` ‚Üí `00 48 00 65 00 6C 00 6C 00 6F` (10 –±–∞–π—Ç–æ–≤), `–ü—Ä–∏–≤–µ—Ç` ‚Üí `04 1F 04 40 04 38 04 32 04 35 04 42` (12 –±–∞–π—Ç–æ–≤).

## 2.3 Subword Tokenization
While byte-level tokenization can alleviate the out-of-vocabulary issues faced by word-level tokenizers, tok-
enizing text into bytes results in extremely long input sequences. This slows down model training, since a sentence with 10 words might only be 10 tokens long in a word-level language model, but could be 50 or
more tokens long in a character-level model (depending on the length of the words). Processing these longer
sequences requires more computation at each step of the model. Furthermore, language modeling on byte
sequences is diÔ¨Äicult because the longer input sequences create long-term dependencies in the data.
